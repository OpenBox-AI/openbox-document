# Getting Started with Cursor

Add OpenBox governance to [Cursor IDE](https://cursor.com). Every agent action — prompts, file reads, shell commands, and MCP tool calls — is evaluated against your policies automatically.

## Prerequisites

- [Node.js](https://nodejs.org/) v18+
- [Cursor IDE](https://cursor.com) v2.5+
- An OpenBox API key — get one at [platform.openbox.ai](https://platform.openbox.ai)

## Install

```bash
git clone https://github.com/OpenBox-AI/cursor-hooks.git
cd cursor-hooks
npm install && npm run build
npm run install-hooks -- --key obx_live_your_key
```

Restart Cursor (`Cmd+Q` → reopen). All agent actions are now governed.

## Verify

```bash
tail -f ~/.cursor-hooks/hook.log
```

Send any prompt in Cursor — you should see events appear in the log.

## Configure on the dashboard

Go to **Agent → Authorize** on the OpenBox dashboard and set up guardrails per activity type:

| Activity type | What to configure |
|--------------|-------------------|
| `PromptSubmission` | PII detection, toxicity, ban words, content filter |
| `FileRead` | PII/secret redaction on file content |
| `ShellExecution` | Command policies (allowlist/blocklist) |
| `MCPToolCall` | Tool policies, MCP server allowlists |
| `MCPToolResponse` | PII redaction on tool output |

## What gets governed

| Agent action | What happens |
|-------------|-------------|
| User submits a prompt | Checked for PII, toxicity, banned content |
| Agent reads a file | Secrets and PII are redacted before the agent sees them |
| Agent runs a shell command | Evaluated against your command policies |
| Agent calls an MCP tool | Checked against tool allowlists |
| MCP tool returns data | PII is redacted from the response |
| Agent responds | Response is scored for goal alignment |

## Verdicts

| Verdict | What you see in Cursor |
|---------|----------------------|
| **Allow** | Action proceeds normally |
| **Constrain** | Sensitive content is redacted, action proceeds |
| **Require approval** | Action is blocked until approved on the dashboard |
| **Block** | Action is blocked with a policy message |
| **Halt** | Action is blocked and the session is terminated |

## Settings

Edit `~/.cursor-hooks/config.json` to change behavior:

| Setting | Default | Description |
|---------|---------|-------------|
| `GOVERNANCE_POLICY` | `fail_open` | `fail_open` allows actions when OpenBox is unreachable. `fail_closed` blocks them. |
| `VERBOSE` | `false` | Enable detailed logging |
| `DRY_RUN` | `false` | Allow all actions without calling OpenBox |

## Uninstall

```bash
rm ~/.cursor/hooks.json
rm -rf ~/.cursor-hooks
```

Restart Cursor.
