# Getting Started with Cursor

OpenBox integrates with Cursor via the official [hooks system](https://cursor.com/docs/hooks) — your IDE workflows stay exactly as they are while every agent action is governed, scored, and auditable.

## What to expect

- Install with a single command — no code changes to your Cursor setup
- Every prompt, file read, shell command, and MCP tool call is governed before execution
- Human-in-the-loop approvals for sensitive operations
- Full audit trail

## Prerequisites

- [Node.js](https://nodejs.org/) v18+
- [Cursor IDE](https://cursor.com)
- An OpenBox API key (`obx_live_xxx`)

## Install

Clone the integration and install dependencies:

```bash
git clone https://github.com/OpenBox-AI/cursor-hooks.git
cd cursor-hooks
npm install
```

Register the hooks with your API key and restart Cursor:

### Global install (all workspaces)

Set your API key and run the installer:

<Tabs groupId="os">
<TabItem value="mac-linux" label="macOS / Linux">

```bash
export OPENBOX_API_KEY=obx_live_your_key
npm run install-hooks
```

</TabItem>
<TabItem value="windows" label="Windows">

**PowerShell:**

```powershell
$env:OPENBOX_API_KEY="obx_live_your_key"; npm run install-hooks
```

**Command Prompt:**

```cmd
set OPENBOX_API_KEY=obx_live_your_key && npm run install-hooks
```

</TabItem>
</Tabs>

Or pass the key as a parameter (the `--` before `--key` is required by npm):

```bash
npm run install-hooks -- --key obx_live_your_key
```

Restart Cursor — all agent actions are now governed.

This writes to `~/.cursor/hooks.json` so every Cursor workspace is governed automatically.

### Project-level install

To scope hooks to a single project instead of all workspaces:

<Tabs groupId="os">
<TabItem value="mac-linux" label="macOS / Linux">

```bash
export OPENBOX_API_KEY=obx_live_your_key
npm run install-hooks -- --project
```

</TabItem>
<TabItem value="windows" label="Windows">

**PowerShell:**

```powershell
$env:OPENBOX_API_KEY="obx_live_your_key"; npm run install-hooks -- --project
```

**Command Prompt:**

```cmd
set OPENBOX_API_KEY=obx_live_your_key && npm run install-hooks -- --project
```

</TabItem>
</Tabs>

Or pass the key as a parameter:

```bash
npm run install-hooks -- --project --key obx_live_your_key
```

Restart Cursor — all agent actions are now governed.

This writes to `.cursor/hooks.json` in the current directory.

### Manual install

Copy `hooks.json` to `~/.cursor/hooks.json` and replace `__HOOK_PATH__` with the absolute path to `dist/hook-handler.js`.

## Configuration

Create or edit `~/.cursor-hooks/config.json`:

```json
{
  "OPENBOX_API_KEY": "obx_live_your_key",
  "OPENBOX_ENDPOINT": "https://core.openbox.ai",
  "GOVERNANCE_POLICY": "fail_open"
}
```

See the [Developer Guide — Configuration](/developer-guide/cursor#configuration) for all available settings.

## Verify it works

Open a Cursor workspace and start an agent conversation. You should see governance activity on the OpenBox dashboard.

To confirm hooks are firing locally:

<Tabs groupId="os">
<TabItem value="mac-linux" label="macOS / Linux">

```bash
tail -f ~/.cursor-hooks/hook.log
```

</TabItem>
<TabItem value="windows" label="Windows">

Open `%USERPROFILE%\.cursor-hooks\hook.log` in any text editor, or stream it with PowerShell:

```powershell
Get-Content $HOME\.cursor-hooks\hook.log -Wait -Tail 20
```

</TabItem>
</Tabs>

## Example: PII detection

When a prompt contains personally identifiable information, OpenBox blocks it before the agent processes it:

```
add seed data for testing: insert a user with name John Smith
and email jsmith@gmail.com into the users table
```

The PII guardrail detected the name and email address in the prompt and halted the submission.
